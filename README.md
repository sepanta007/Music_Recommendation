## Overview

This system processes a catalog of **28,372 songs spanning 70 years (1950–2019)** to deliver ranked, diverse playlists based on a weighted multi-dimensional similarity score. It was designed with the following principles:

- **Genre compatibility** is the most important signal
- **Audio characteristics** (danceability, energy, valence, etc.) strongly influence enjoyment
- **Diversity constraints** prevent playlist monotony (max 1 song per artist)
- **Transparency**: every recommended song includes its similarity score and full metadata

## Algorithm

### Data Preparation (`preperation.py`)

The preparation script runs a three-stage pipeline:

1. **Genre ID Assignment** — Extracts all unique genres from `data_w_genres.csv`, sorts them alphabetically, and assigns sequential numeric IDs.
2. **Artist–Genre Mapping** — Maps each artist to their set of genre IDs. Artists not found in the external source fall back to default mappings (rock, blues, pop, country, jazz, reggae, hip hop).
3. **Music Data Processing** — Merges the primary dataset with genre mappings, assigns unique `track_id` and `artist_id` values, and reduces dimensionality by extracting only the **top 3 topics** and **top 3 audio features** per song (ranked by score).

### Recommendation Engine (`recommendation.py`)

The system computes a **weighted hybrid similarity score** between the query song and every other song in the catalog across five dimensions:

| Criterion | Weight | Description |
|-----------|:------:|-------------|
| 🎸 Genre | **6** | Genre set overlap (Jaccard-style, penalizing size mismatches) |
| 🎧 Audio Features | **5** | Matching top-3 audio feature indices + bonuses for exact value matches |
| 👤 Artist | **3** | Bonus for songs by the same artist |
| 📝 Topic | **3** | Overlap in top-3 lyrical topic categories |
| 📅 Time | **2** | Release era proximity using logistic decay |

**Total similarity score:**

```
score = 6×genre_sim + 5×feature_sim + 3×artist_sim + 3×topic_sim + 2×time_sim
```

#### Temporal Similarity Detail

The temporal component uses a **logistic decay function** for a smooth, natural penalty on era differences:

```python
logistic_sim = 1 / (1 + exp(0.1 × |year_u - year_v|))
time_sim = log(|year_u - year_v| + 1) × logistic_sim   # if years differ
time_sim = logistic_sim                                  # if same year
```

#### Playlist Construction

1. Score all songs against the query
2. Sort by score (descending)
3. Greedily add songs to the playlist, enforcing **max 1 song per artist**
4. Stop when 100 songs are collected or candidates are exhausted

## Setup & Usage

### Prerequisites

```bash
pip install pandas
```

### Step 1 — Prepare the data *(one-time)*

```bash
python src/preperation.py
```

Expected output:

```
Genre-ID mapping saved to data/genres_with_ids.csv
Artist-genre mapping saved to data/artist_genre_mapping.csv
Processed data saved to data/final_processed_music_data.csv
```

### Step 2 — Generate a playlist

```bash
python src/recommendation.py
```

You will be prompted to enter a track ID:

```
Entrez l'ID du morceau de départ (entre 1 et 28372) : 42
Playlist saved to data/generated_playlist_from_<artist>_<song>.csv
```

### Step 3 — View results

```bash
cat data/generated_playlist_from_<artist>_<song>.csv
```

## Output Format

The generated playlist is saved as a CSV file at:

```
data/generated_playlist_from_<artist>_<song>.csv
```

| Column | Description |
|--------|-------------|
| `Score` | Similarity score (empty for the query song) |
| `Track ID` | Unique track identifier |
| `Artist` | Artist name |
| `Song` | Track name |
| `Genre` | List of genre IDs |
| `Release Date` | Release year |
| `Topic_1/2/3` | Top 3 lyrical topics as `(index, score)` tuples |
| `Feature_1/2/3` | Top 3 audio features as `(index, score)` tuples |

The first row is always the **query song** (no score). Subsequent rows are the top 100 recommendations sorted by similarity score in descending order.

## Features

- 🎯 **Multi-Dimensional Similarity** — Combines 5 independent metrics for robust recommendations
- 🎸 **Genre-First Approach** — Genre compatibility weighted highest for music-relevant results
- 📊 **28,372-Song Catalog** — Covers 70 years of music history (1950–2019)
- 🌍 **Artist Diversity** — At most 1 song per artist per playlist
- 📈 **Transparent Scoring** — Every recommendation includes its full similarity score
- ⏳ **Temporal Awareness** — Logistic decay function naturally prefers songs from similar eras
- 🔧 **Minimal Dependencies** — Only `pandas` required beyond the Python standard library
- 📁 **CSV-Based Pipeline** — Portable and Excel-compatible, no database required
